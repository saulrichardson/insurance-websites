# Deployment Hardening

- date: 2026-06-10

## Context

The repo was deployable locally, but production still had deployment drift:
project runtime settings were inconsistent, dependency versions could float,
the personal site's canonical metadata did not match its public redirect host,
and `go.tracyzhanginsurance.com` was not attached to the insurance deployment.

## Record

Treat deployment as a first-class contract for both public websites:

- Production runtime is Node 24.x for both Vercel projects.
- Root dependency resolution is pinned by `pnpm-lock.yaml`, exact manifest
  versions, and frozen installs.
- `apps/tracy-zhang` deploys from the monorepo root through the Vercel project
  `insurance-websites` using the remote Production build path. Do not ship a
  local prebuilt production bundle for this app, because root `.env.local` is a
  local-only file and must not define public production metadata.
- `apps/tracy-zhang-insurance` deploys through the staged app-at-root package
  generated by `scripts/stage-tracy-zhang-insurance-vercel.mjs`. The staging
  package carries the root package manager, Node engine, pnpm settings, root
  lockfile, and imported shared packages, then derives its staged lockfile
  offline before a frozen install.
- `go.tracyzhanginsurance.com` belongs on the canonical insurance Vercel
  project as a DNS-only Vercel `A` target, alongside the canonical and local
  insurance domains.
- The personal site's production canonical URL is
  `https://www.tracyzhang.com`, matching the apex-to-www redirect.

## Evidence

- Vercel project settings report Node `24.x` for `insurance-websites` and
  `san-marino-insurance`.
- Personal production deployment:
  `insurance-websites-hk2qpnpd6-saulrichardsons-projects.vercel.app`.
- Insurance production deployment:
  `san-marino-insurance-rgy3njlle-saulrichardsons-projects.vercel.app`.
- Live smoke checks returned HTTP 200 for `tracyzhanginsurance.com`,
  `www.tracyzhanginsurance.com`, `sanmarinoinsurance.com`,
  `www.sanmarinoinsurance.com`, `lapalmainsurance.com`,
  `www.lapalmainsurance.com`, `go.tracyzhanginsurance.com`, and
  `www.tracyzhang.com`.
- `https://tracyzhang.com/` redirects to `https://www.tracyzhang.com/`, and
  the rendered canonical link, robots sitemap URL, and sitemap locs now use
  `https://www.tracyzhang.com`.
- Invalid insurance lead API smoke input returns HTTP 400 with structured
  validation errors.

## Future Guidance

Keep local env files local. If a production metadata value changes, update
Vercel Production env and deploy through the documented production path. If the
insurance Vercel project is later converted into a monorepo-root project, keep
the same invariants: Node 24.x, pnpm, exact versions, frozen lockfile install,
and a smoke check across every attached domain after deployment.
